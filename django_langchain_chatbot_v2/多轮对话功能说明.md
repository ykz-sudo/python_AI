# 多轮对话功能说明

## 功能概述

已成功为Django聊天机器人添加了多轮上下文对话功能。现在系统能够记住对话历史，并在回答问题时考虑之前的对话内容，提供更加连贯和智能的对话体验。

## 主要改动

### 1. 数据库模型 (models.py)

新增了两个模型来管理对话会话：

- **ChatSession**: 会话模型
  - `session_id`: UUID类型的唯一会话标识符
  - `user`: 用户标识（默认为'anonymous'）
  - `created_at`: 会话创建时间
  - `updated_at`: 会话最后更新时间

- **ChatMessage**: 消息模型
  - `session`: 关联的会话（外键）
  - `role`: 消息角色（'user' 或 'assistant'）
  - `content`: 消息内容
  - `created_at`: 消息创建时间

### 2. RAG模块 (rag.py)

更新了 `answer_with_rag` 函数以支持对话历史：

- 新增 `chat_history` 参数，接收历史对话列表
- 在生成回答时，将对话历史和知识库内容一起传递给LLM
- 保留最近10条消息（5轮对话）作为上下文
- 使用系统提示词指导AI结合历史对话和知识库内容回答

### 3. 视图层 (views.py)

更新了 `api_chat` 函数：

- 接收并处理前端传来的 `session_id`
- 如果没有会话ID或会话不存在，自动创建新会话
- 保存每条用户消息和AI回复到数据库
- 在调用RAG时传入对话历史
- 返回响应时包含 `session_id`，供前端后续请求使用

### 4. 前端JavaScript (chatbot_app.js)

增强了前端交互功能：

- 使用 `sessionId` 变量存储当前会话ID
- 在发送消息时自动携带会话ID
- 接收并保存服务器返回的会话ID
- 新增 `clearChat()` 函数，用于清空对话并开始新会话

### 5. 前端HTML (chat.html)

- 更新页面标题，标明支持多轮对话
- 添加"清空对话"按钮，允许用户重新开始对话

## 使用方法

### 1. 执行数据库迁移

在项目根目录下运行：

```bash
python manage.py migrate
```

这将创建 `ChatSession` 和 `ChatMessage` 表。

### 2. 启动服务器

```bash
python manage.py runserver
```

### 3. 测试多轮对话

访问 http://localhost:8000/ 并进行对话测试：

**示例对话：**

```
用户: 你们公司的营业时间是什么？
AI: [根据知识库回答营业时间]

用户: 那周末呢？
AI: [AI会理解"周末"是在询问营业时间，结合上下文回答]

用户: 我想预约明天下午
AI: [AI会理解用户想预约服务，并结合之前的对话提供帮助]
```

点击"清空对话"按钮可以开始新的会话。

## 技术特点

1. **会话持久化**: 所有对话历史都保存在数据库中，可用于后续分析
2. **上下文感知**: AI能够理解代词和省略的内容，提供更自然的对话体验
3. **历史限制**: 只保留最近10条消息作为上下文，避免token过多
4. **会话管理**: 支持多个用户同时使用，每个会话独立管理
5. **无缝集成**: 保留了原有的联系方式提取和负面反馈检测功能

## 数据库表结构

### chatbot_app_chatsession
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BigAutoField | 主键 |
| session_id | UUIDField | 会话唯一标识 |
| user | CharField(128) | 用户标识 |
| created_at | DateTimeField | 创建时间 |
| updated_at | DateTimeField | 更新时间 |

### chatbot_app_chatmessage
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BigAutoField | 主键 |
| session_id | ForeignKey | 关联会话 |
| role | CharField(16) | 角色(user/assistant) |
| content | TextField | 消息内容 |
| created_at | DateTimeField | 创建时间 |

## 注意事项

1. 确保已安装所有依赖包（见 requirements.txt）
2. 如果遇到 `langchain.text_splitter` 导入错误，已修改为使用 `langchain_text_splitters`
3. 对话历史会占用数据库空间，建议定期清理旧会话
4. 可以根据需要调整保留的历史消息数量（当前为10条）
