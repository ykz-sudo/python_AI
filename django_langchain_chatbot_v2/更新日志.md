# 更新日志

## 2026-01-30 - 多轮对话功能上线

### 新增功能

#### 1. 多轮上下文对话 ⭐
- **会话管理**: 每个用户对话自动创建唯一会话ID
- **上下文记忆**: AI能够记住之前的对话内容，理解代词和省略的表达
- **历史限制**: 保留最近10条消息作为上下文，避免token过多
- **会话持久化**: 所有对话历史保存在数据库中

#### 2. 会话历史查看页面 📋
- 访问 `/sessions/` 查看所有会话记录
- 显示每个会话的完整对话历史
- 统计总会话数和消息数
- 美观的UI界面，区分用户和AI消息

#### 3. 清空对话功能 🔄
- 前端新增"清空对话"按钮
- 点击后清空当前会话，开始新对话
- 不影响数据库中已保存的历史记录

#### 4. 会话清理管理命令 🧹
- 新增 `cleanup_old_sessions` 管理命令
- 可定期清理指定天数之前的旧会话
- 支持模拟运行模式（--dry-run）

### 数据库变更

#### 新增表

**chatbot_app_chatsession** - 会话表
```sql
- id: BigAutoField (主键)
- session_id: UUIDField (唯一会话标识)
- user: CharField(128) (用户标识)
- created_at: DateTimeField (创建时间)
- updated_at: DateTimeField (更新时间)
```

**chatbot_app_chatmessage** - 消息表
```sql
- id: BigAutoField (主键)
- session_id: ForeignKey (关联会话)
- role: CharField(16) (user/assistant)
- content: TextField (消息内容)
- created_at: DateTimeField (创建时间)
```

### 使用示例

#### 多轮对话示例
```
用户: 你们公司的营业时间是什么？
AI: 我们的营业时间是周一至周五 9:00-18:00

用户: 周末也营业吗？
AI: [理解"周末"是在问营业时间] 周末我们不营业...

用户: 那我想预约明天下午
AI: [理解用户想预约服务] 好的，请问您需要预约什么服务？
```

#### 清理旧会话
```bash
# 删除30天前的会话
python manage.py cleanup_old_sessions --days 30

# 模拟运行
python manage.py cleanup_old_sessions --days 30 --dry-run
```
